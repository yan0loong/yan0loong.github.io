Centos7搭建Hadoop集群

Linux Hadoop
大数据
发布日期:   2020-11-09
文章字数:   1.4k
阅读时长:   6 分
阅读次数:  

1. 本文集群搭建分布及工具版本

1. 工具及其版本

jdk1.8.0_231
hadoop-2.7.7

2. 集群部署规划

master slave1 slave2
IP 192.168.28.10 192.168.28.11 192.168.28.12
内存 4G 4G 4G
HDFS NN/DN DN 2NN/DN
YARN NM RN/NM NM

2. 安装三台虚拟机及基本配置

1. 修改用户名以及对应IP

配置主机名
# vi /etc/hostname
改为master
另外那个虚拟机也按同样的设置，hostname文件内容分别改为 slave1、slave2。设置完后每台虚拟机都要重启，重启生效。
配置完成后使用ping命令检查这3个机器是否相互ping得通，以slave1为例:
# ping -c 3 slave1

配置主机名映射
# vi /etc/hosts
配置如下：
192.168.28.10 master
192.168.28.11 slave1
192.168.28.12 slave2

2. 进行SSH免密互登设置

在命令执行过程中敲击两遍回车，然后在/root/.ssh文件下生成id_dsa id_dsa.pub
在该文件下建立一个authorized_keys文件，将id_dsa.pub文件内容拷贝到authorized_keys文件中
另外两个虚拟机也进行相同的操作，并将自己的id_dsa.pub文件内容拷贝到其他两个虚拟机/root/.ssh下的authorized_keys文件中
通过命令 # ssh slave1 看是否可以免密登录，可以通过exit退出

3. 安装Java环境

1. 解压jdk-8u231-linux-x64.tar.gz文件

将下载的 jdk-8u231-linux-x64.tar.gz 的包上传到 master 环境的 /usr/local/src (我习惯讲所有东西放这个路径，可以自己建一个专门的文件夹来放这些包)路径下，然后 tar -xzvf jdk-8u231-linux-x64.tar.gz 解压到当前目录下，会生成一个 jdk1.8.0_231 文件夹

2. 配置 jdk 环境变量

修改 /etc/profile 这个文件，增加如下配置：
export JAVA_HOME=/usr/local/src/jdk1.8.0_231
export CLASSPATH=$:CLASSPATH:$JAVA_HOME/lib/
export PATH=$PATH:$JAVA_HOME/bin
修改后保存，然后再执行 source /etc/profile 命令使其生效
然后执行 java -version 命令测试是否按照成功

3. 安装Hadoop环境

1. 解压hadoop-2.7.7.tar.gz文件

将下载的 hadoop-2.7.7.tar.gz 的包上传到 master 环境的 /usr/local/src 路径下，然后 tar -xzvf hadoop-2.7.7.tar.gz 解压到当前目录下，会生成一个 hadoop-2.7.7 文件夹

2. 配置hadoop环境变量

修改 /etc/profile 这个文件，增加如下配置：
export HADOOP_HOME=/usr/local/src/hadoop-2.7.7
修改 之前的 PATH 为如下：
export PATH=$PATH:$JAVA_HOME/bin:$HADOOP_HOME/bin:$HADOOP_HOME/sbin
修改后保存，然后再执行 source /etc/profile 命令使其生效
然后执行 hadoop version 命令测试是否按照成功

3. 核心配置文件

1. 配置 masters 和 slaves 文件

如果没有这两个文件，自己创建就行。只需要将指定域名配置在这两个文件中，masters 文件中配置 master 节点的主机名，slaves 文件配置 slave 节点的主机名。
masters 文件中添加:
master
slaves 文件中添加:
master
slave1
slave2

2. 核心配置文件

配置 core-site.xml
在该文件下编辑如下配置：
<!-- 指定HDFS中NameNode的地址 -->
<property>
    <name>fs.defaultFS</name>
    <value>hdfs://master:9000</value>
</property>

<!-- 指定Hadoop运行时产生文件的存储目录 -->
<property>
    <name>hadoop.tmp.dir</name>
    <value>/usr/local/src/hadoop-2.7.7/tmp</value>
</property>

3. HDF配置文件

配置 hadoop-env.sh
添加java环境变量 export JAVA_HOME=/usr/local/src/jdk1.8.0_231

配置 hdfs-site.xml
在该文件中编写如下配置
<property>
    <name>dfs.replication</name>
    <value>3</value>
</property>

<!-- 指定Hadoop辅助名称节点主机配置 -->
<property>
    <name>dfs.namenode.secondary.http-address</name>
    <value>slave2:50090</value>
</property>

4. YARN 置文件

配置 yarn-env.sh
添加java环境变量 export JAVA_HOME=/usr/local/src/jdk1.8.0_231

配置 yarn-site.xml
在该文件中增加如下配置
<!-- reducer获取数据的方式 -->
<property>
    <name>yarn.nodemanager.aux-services</name>
    <value>mapreduce_shuffle</value>
</property>

<!-- 指定YARN的ResourceManager的地址 -->
<property>
    <name>yarn.resourcemanager.hostname</name>
    <value>slave1</value>
</property>

5. MapReduce 配置文件

配置 mapred-env.sh
添加java环境变量 export JAVA_HOME=/usr/local/src/jdk1.8.0_231

配置 mapred-site.xml
在该文件中增加如下配置
<!-- 指定mr运行在yarn上 -->
<property>
    <name>mapreduce.framework.name</name>
    <value>yarn</value>
</property>

4. 分发配置文件

1. 分发 /etc/hosts 配置文件

将 master上的 /etc/hosts 文件分发到 slave1、slave2, 以下列子以 slave1 为列
# scp -r /etc/hosts root@slave1:/etc/

2. 分发 /etc/profile 配置文件

将 master上的 /etc/profile 文件分发到 slave1、slave2, 以下列子以 slave1 为列
# scp -r /etc/profile root@slave1:/etc/
分发后分别在三台虚拟机上执行 source /etc/profile 使其配置生效

3. 分发 java 文件

将 master上的 /usr/local/src/jdk1.8.0_231 文件分发到 slave1、slave2, 以下列子以 slave1 为列
# scp -r /usr/local/src/jdk1.8.0_231 root@slave1:/usr/local/src/

4. 分发 hadoop 文件

将 master上的 /usr/local/src/hadoop-2.7.7 文件分发到 slave1、slave2, 以下列子以 slave1 为列
# scp -r /usr/local/src//usr/local/src/hadoop-2.7.7 root@slave1:/usr/local/src/

5. 启动集群

如果集群是第一次启动，需要格式化 NameNode
# bin/hdfs namenode -format
启动集群
# sbin/start-all.sh
